# Senior Developer Review Checklist

## 1. Code Quality
- [ ] Consistent modular structure (TOP / SEL / MAIN)
- [ ] AUTHORITY-CHECK implemented at the beginning
- [ ] Field-Symbols used for performance
- [ ] Proper error handling and meaningful messages
- [ ] BAPI usage instead of direct updates (where applicable)

## 2. Architecture & Design
- [ ] Clear separation of concerns
- [ ] Integration points (MM-FI, SD-WM) properly handled
- [ ] Custom development justified (Fit-to-Standard first)
- [ ] CDS Views used for analytical layer

## 3. Security & Authorization
- [ ] Roles and SoD concept considered
- [ ] Authorization objects properly checked
- [ ] No over-authorization

## 4. Documentation & Visualization
- [ ] README.md is clear and comprehensive
- [ ] Diagrams (Mermaid) are up-to-date and useful
- [ ] Test cases documented
- [ ] Design decisions explained

## 5. Overall Impression
- [ ] Project shows strong process thinking
- [ ] Ready for real business discussion
- [ ] Maintainable and scalable
